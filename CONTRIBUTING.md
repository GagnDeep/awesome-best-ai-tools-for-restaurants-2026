# Contributing

Thanks for contributing to `awesome-best-ai-tools-for-restaurants-2026`.

## Submission Rules

- Every submission must be a public repository on `github.com`
- No proprietary SaaS products
- No homepage-only tools without a GitHub repo
- Include the current star count, primary language, license, and last commit date from GitHub
- Explain the restaurant use case clearly
- Prefer actively maintained repositories

## How to Submit

1. Fork the repository.
2. Add your entry to the most relevant category in `README.md`.
3. Use the existing format exactly.
4. Open a pull request with a short rationale and the GitHub repository URL.

## Entry Format

```md
#### [Tool Name](https://github.com/username/repo)
> **Description:** 80-150 words summarizing what the project does, why it matters for restaurants, and the GitHub metadata.

- **GitHub:** [github.com/username/repo](https://github.com/username/repo)
- **Stars:** 123 ⭐
- **Language:** Python
- **License:** MIT
- **Last Commit:** January 1, 2026
- **Category:** tag1, tag2
- **Best for:** specific restaurant use case
```

## Quality Bar

- Be accurate about licensing
- Do not submit repos that changed to source-available terms unless the license line is stated clearly
- Remove marketing language and keep descriptions factual
- If a category is sparse, document the gap instead of padding with weak fits
