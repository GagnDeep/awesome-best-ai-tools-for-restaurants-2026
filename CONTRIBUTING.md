# Contributing

Thanks for contributing to `awesome-best-ai-tools-for-restaurants-2026`.

## Submission Rules

To keep this repository consistent, every submission must meet all of the following:

1. The tool must have a public repository on `github.com`.
2. The repository must contain usable source code, not just marketing pages, issue trackers, or mirrors without substance.
3. The tool must be relevant to restaurant workflows such as ordering, menu OCR, voice automation, forecasting, recommendations, inventory, customer support, or operations.
4. No proprietary SaaS products, closed hosted services, or "free tier" tools without public source code.
5. Include the repo's primary language, license, and a short explanation of why it belongs in this list.

## Preferred Pull Request Format

Please include:

- Repository name and URL
- One-paragraph description
- License
- Primary language
- Latest visible commit date
- Suggested category
- Restaurant-specific use case

## Quality Bar

- Be honest about maintenance status.
- Be honest about category gaps.
- Prefer actively maintained repositories when possible.
- If a repo has a custom or source-available license, call that out clearly in the PR.

## Style Notes

- Keep descriptions specific.
- Do not add tools that do not link directly to a GitHub repository.
- Do not add duplicates, forks without material differentiation, or unrelated generic AI repos.

## Reporting Issues

Open an issue if:

- A repository has been deleted or made private
- A license entry is wrong
- A tool is no longer appropriate for restaurant use
- A better open source GitHub alternative should replace an existing entry
